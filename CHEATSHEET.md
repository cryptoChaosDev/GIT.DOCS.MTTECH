## 🔑 Создание GitHub токена

**1. Открыть:** GitHub.com → Settings (правый верх)
**2. Перейти:** Developer settings (внизу слева)
**3. Выбрать:** Personal access tokens → Tokens (classic)
**4. Создать:** Generate new token (classic)
**5. Заполнить:**
   - Note: `Git Docs Bot`
   - Expiration: `90 days`
   - ☑️ **repo** (все пункты!)
**6. Сгенерировать:** Generate token
**7. СКОПИРОВАТЬ СРАЗУ!** (показывается один раз)

## ⚠️ ВАЖНО! Правила работы

| ✅ ДЕЛАТЬ | ❌ НЕ ДЕЛАТЬ |
|----------|-------------|
| Блокировать перед работой | Работать без блокировки |
| Разблокировать после работы | Оставлять блокировку |
| Обновлять репозиторий вначале | Работать со старой версией |
| Загружать изменения в конце | Не синхронизировать |
| Хранить токен в секрете | Передавать токен другим |

## 🆘 Быстрое решение проблем

| Ошибка | Что делать |
|--------|-----------|
| `Репозиторий не настроен` | ℹ️ О репозитории → ⚙️ Настроить |
| `Нет доступа` | Проверить токен и права |
| `Документ заблокирован` | Подождать разблокировки |
| `Файл не загружается` | Проверить формат (.docx) |
| `Конфликт при загрузке` | 🔄 Обновить → повторить |
| `Токен истек` | Создать новый токен |

## 📊 Статусы блокировки

| Значок | Статус | Действие |
|--------|--------|----------|
| 🟢 | Свободен | Можно блокировать |
| 🔒 Вы | Заблокирован вами | Можно редактировать |
| 🔒 Имя | Заблокирован другим | Ждать разблокировки |

---

## 💡 Полезные советы

1. **Синхронизация**: Обновляйтесь 🔄 в начале дня
2. **Блокировка**: Всегда блокируйте 🔒 перед работой
3. **Разблокировка**: Не забывайте разблокировать 🔓
4. **Коммуникация**: Договаривайтесь с коллегами о работе
5. **Резервные копии**: Git хранит всю историю изменений
